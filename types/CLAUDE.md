# types/ — Agent Context

Shared TypeScript type definitions. Currently one file: `fact.ts`.

## `fact.ts`

### `Fact`
The top-level saved item. Stored in AsyncStorage and Supabase `facts` table.

```ts
interface Fact {
  id: string;           // UUID
  question: string;     // The user's original question
  answer: string;       // The LLM's full answer text
  mnemonic?: string;    // Optional memory hook (LLM-generated)
  createdAt: Date;      // When it was saved (Date after hydration, string in JSON)
  card: Card;           // FSRS card state for legacy review (Date fields after hydration)
  atomicCards?: AtomicCard[];  // New: individual Q&A flashcards split from the answer
}
```

**`card` vs `atomicCards`**: A Fact always has `card` (legacy FSRS state), but new facts also have `atomicCards`. The `card` on the Fact itself is not used for review when `atomicCards` exist — each `AtomicCard` has its own `card`. The top-level `card` on new facts is essentially unused for scheduling.

### `AtomicCard`
One flashcard derived from a Fact. Generated by `generateAtomicCards()` in `services/llm.ts`.

```ts
interface AtomicCard {
  id: string;      // UUID
  front: string;   // The question side
  back: string;    // The answer side
  card: Card;      // FSRS card state (this is what drives scheduling)
  mnemonic?: string;
}
```

### `ReviewItem`
Union type used by the review/recall screen. Abstracts over both legacy and atomic review.

```ts
interface ReviewItem {
  fact: Fact;            // Always present — the parent Fact
  atomicCard?: AtomicCard;  // Present for atomic cards; undefined for legacy facts
  isLegacy: boolean;     // true = review the Fact.card; false = review atomicCard.card
}
```

To get what to display: if `isLegacy`, show `fact.question` / `fact.answer`. Otherwise show `atomicCard.front` / `atomicCard.back`.
To get the card for FSRS scheduling: `isLegacy ? fact.card : atomicCard!.card`.

### `ReviewLog`
One entry per review action. Stored locally for streak + stats calculation.

```ts
interface ReviewLog {
  factId: string;
  atomicCardId?: string;  // Present if the reviewed item was an atomic card
  rating: number;         // FSRS Rating value (1=Again, 2=Hard, 3=Good, 4=Easy)
  reviewedAt: Date;       // Timestamp (Date after hydration)
}
```

## `Card` type
Comes from `ts-fsrs`. Key fields: `due: Date`, `last_review?: Date`, `stability: number`, `difficulty: number`, `state: State`. Always imported from `ts-fsrs` or via `services/srs.ts`.
